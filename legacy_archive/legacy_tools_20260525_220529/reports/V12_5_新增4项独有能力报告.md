## 2. 各功能设计点详述

### 功能 1: 跨论文矛盾检测 (`contradiction_detector.py`)

**核心设计**：
- 覆盖 **3 类矛盾**：数值矛盾（numeric）、机制矛盾（mechanism）、边界条件矛盾（boundary）
- **优先读 limitations/discussion 章节**（沿用 V12 核心设计），非全文扫描
- 输出结构化矛盾列表，含 severity (low/mid/high) + 双方 paper_id + 中文解释
- 支持所有 17 个主题（T01-T17），每个主题有专门的语义检索查询词
- LLM 矛盾检测（`pplx llm extract`，max_tokens=8000，成本约 $0.001/次）

**与 paper-qa 差异**：
- paper-qa 矛盾检测是通用文本比较
- 本模块专注 AI4Science 卡点（限制/讨论章节），区分 3 类矛盾类型，适配领域知识

**CLI**:
```bash
python -m scibot.contradiction_detector --theme T01
python -m scibot.contradiction_detector --theme T12 --top-k 10 --output result.json
```

### 功能 2: 多跳引用追踪 (`citation_chain.py`)

**核心设计**：
- 支持 **深度 2 引用追踪**（可配置），防止循环引用
- 智能参考文献提取：
  - 优先合并多行 [N] 格式引用条目（处理 PDF 换行问题）
  - 处理 Unicode 智能引号（`"..."` = 0x201C/0x201D）
  - 正则提取失败时自动 LLM 兜底
- **语料库内查找**：Jaccard 词级相似度（阈值 0.25），在 25 篇已有论文中检索匹配
- **共同祖先检测**：图遍历找被多条路径同时引用的论文
- 输出：完整引用树 + 共同祖先 + 在库/不在库标注

**与 paper-qa 差异**：
- paper-qa **不提供引用链追踪**
- 本模块是 paper-qa 完全没有的能力

**CLI**:
```bash
python -m scibot.citation_chain --paper 01KR7T0X3W40FACCMYC42QXFHV
python -m scibot.citation_chain --paper 01KR7T0VQ0VWCDTX5SN9B4BEVH --depth 2
```

### 功能 3: 元规律→论文反查 (`principle_to_papers.py`)

**核心设计**：
- 输入：MP1-MP4 任一元规律 ID
- 流程：从元规律 JSON 获取覆盖主题 → 每个主题拉 limitations chunks → LLM 按"本源严重度"打分
- 输出论文按严重度（0-1.0）降序排列，含原文证据 + 中文解释
- LLM 失败时自动 fallback 到 first_principles_results.json 的现有数据
- 覆盖 **17 个主题 / 4 个元规律**，数据完全来自 V12 已有语料

**样例输出（MP1: 维度灾难）**：
```json
{
  "principle": "维度灾难与非凸地形搜索瓶颈",
  "papers_ranked_by_severity": [
    {
      "paper_id": "01KR7T0VR8AJJY23SJHDCJW1GJ",
      "paper_title": "Broadband thermal imaging using meta-optics",
      "severity": 0.95,
      "evidence": "sampling complexity increases exponentially with the number of...",
      "how_principle_manifests": "该论文直接指出了高维设计的数学困境..."
    }
  ]
}
```

**与 paper-qa 差异**：
- paper-qa 是被动问答，无元规律概念
- 本模块的"元规律"是 V12 独有的概念层，是 paper-qa 完全没有的抽象

**CLI**:
```bash
python -m scibot.principle_to_papers --principle MP1
python -m scibot.principle_to_papers --principle MP3 --top-n 5
```

### 功能 4: 新论文增量分析 (`incremental_analysis.py`)

**核心设计**：
- 输入：arXiv URL 或本地 PDF 路径
- 全流程自动化：
  1. arXiv PDF 自动下载（支持 `/abs/` 和 `/pdf/` URL 格式）
  2. PDF 解析（pymupdf 优先，pdftotext 兜底）
  3. 词级快速语义匹配：论文文本 × 17主题关键词（Jaccard 相似度）
  4. 词级元规律预匹配（MP1-MP4）
  5. LLM 精细分析：主题确认 + 元规律严重度 + 5项追问 + 知识关系判断
- **知识关系分类**：reinforces / counterexample / new_direction / unclear
- **5项追问**完全沿用 V12 first_principles 范式（What/How/Why/Where/Predict）

**样例输出（半监督学习论文 SelfMatch）**：
- 最贴近主题: T15（离线与跨域RL的分布漂移修正），对齐分 0.68
- 最严重元规律: MP2（流形假设失真），严重度 0.85
- 知识关系: reinforces（强化T08的流形相关论述）
- 5项追问：完整分析流形假设失真的机制、信息论本源、跨域对偶、可证伪预测

**与 paper-qa 差异**：
- paper-qa 只能对已索引论文问答
- 本模块将**任意新论文**自动接入 V12 知识图谱，输出结构化增量报告

**CLI**:
```bash
python -m scibot.incremental_analysis --arxiv https://arxiv.org/abs/2401.12345
python -m scibot.incremental_analysis --pdf /path/to/paper.pdf --output result.json
```

## 4. LLM 成本估算

| 功能 | max_tokens | 每次约成本 | 备注 |
|------|-----------|-----------|------|
| contradiction_detector | 8000 | ~$0.001 | 含 5000 chars 上下文 |
| citation_chain | 5000 | ~$0.0005 | 仅 ref 提取 LLM 兜底 |
| principle_to_papers | 8000 | ~$0.001 | 含 3000 chars 上下文 |

**单次全量运行（4功能各1次）估算**：约 $0.004-0.005，远低于 $1 预算。

## 5. 与 paper-qa 差异化对比表（4项新功能）

| 能力维度 | Sci-Bot V12.5 (新增) | paper-qa | 优势来源 |
|---------|---------------------|---------|---------|
| **矛盾检测类型** | 3类分类（数值/机制/边界）+ AI4Science 领域定制 | 通用矛盾检测 | 领域专用分类体系 |
| **矛盾检测来源** | 优先 limitations/discussion 章节 | 全文 flat chunk | V12 核心设计：读限制章节 |
| **引用链追踪** | ✅ 深度2跳 + 共同祖先 + 在库检测 | ❌ 完全不支持 | Sci-Bot 独有 |
| **元规律概念层** | ✅ MP1-MP4，反向从抽象到具体 | ❌ 无元规律概念 | Sci-Bot 独有抽象层 |
| **新论文接入** | ✅ 自动 arXiv 下载 + 5项追问 + 知识图谱接入 | ✅ 支持新论文索引，但只做 QA | Sci-Bot 主动分析 vs paper-qa 被动问答 |
| **知识关系判断** | ✅ reinforces/counterexample/new_direction | ❌ 不做 | Sci-Bot 独有认知层 |
| **本源严重度** | ✅ 0-1.0 量化论文与元规律的体现程度 | ❌ 不做 | Sci-Bot 独有 |
