## 2. N1 — 数据采样策略 + 自适应权重

### 2.1 现象引用

来源:`V11_3_hotfix_root_cause.md` §R7 + V11.4 plan §N1。

```
V11.2 语料(2024-2026):cite_direct = 510
V11.3 语料(2022-2023):cite_direct = 1042
差异倍数:≈ 2×
```

新论文发表后 12-18 个月内,被同期语料引用的可能性极低。在 1000 篇抽样规模下,2024-2026 语料的 cite_direct 信号几乎失效,而 2022-2023 语料信号充足。两批语料若以相同权重合并,新语料 cite_direct 会被系统性低估。

### 2.2 根因

新论文被引周期:论文发表后平均需要 12-18 个月才在同期语料中积累有效引用量。在语料内部引用矩阵中,2024-2026 新语料的 cite_direct 仅 510 条,而 2022-2023 老语料为 1042 条——两者在同等样本量下差距 2 倍。

V11.4 合并语料后(raw + raw_v2 各 1000 篇,总 2000 篇),v1 平均年龄 22.4 个月,v2 平均年龄 42.9 个月,合并平均年龄 32.6 个月。若不加区分直接赋予统一 cite_direct 权重,则忽略了语料年龄结构差异。

### 2.3 V11.4 方案(N1-A)

**新文件**:`echelon/ingest/sampling_strategy.py`

核心函数:`select_pilot_sampling_window(target_size, age_priority)`

```
age_priority = "fresh"       → 6-18 个月窗口(emerging 信号,cite_direct 弱)
age_priority = "established" → 12-36 个月窗口(默认,信号充足)
age_priority = "mature"      → 36-60 个月窗口(cite_direct 强但已知度高)
```

**自适应权重公式**:

```
cite_direct_weight = clip(avg_age_months / 18, 0.3, 1.0)
```

含义:语料平均年龄 ≥ 18 个月时权重上限 1.0;低于 18 个月时按比例降权至最低 0.3,同时提升 co_citation 权重补偿。

**V11.4 实际结果**:
- 合并语料平均年龄 32.6 个月 → cite_direct_weight = clip(32.6/18, 0.3, 1.0) = **1.0**
- co_citation_weight = 0.742(自适应结果)

文档更新:第 14 章 MVP 路线图增加"Pilot 语料采样指南"一节。

## 3. N2 — 物理深度 Path 2 细化 + Path 4 新增

### 3.1 现象引用

来源:`V11_4_plan.md` §N2。

```
V11.3 T11714:Path 2 贡献 60 篇 / 153 篇通过 = 39%
V11.3 Path 2 关键词:仅 SOTA% / dataset / ablation(三词)
```

V11.3 R5 hotfix 用最简关键词集实现 OR 化物理深度门,成功消除了 Optics-only 歧视。但随即暴露两个新问题:Path 2 太宽导致假阳性(任何含"achieves 87%"的句子均通过),且纯理论 ML 论文(无 SOTA 数值但有数学证明)仍被漏判。

### 3.2 根因

V11.3 R5 的三条关键词(SOTA% / dataset / ablation)过于粗粒度:

1. **假阳性**:描述他人工作时也会出现"achieves 87% on COCO",当前规则无法区分"自己结果"与"引用他人"
2. **假阴性**:纯理论 ML 论文(有复杂度证明、收敛保证,无数值表格)无法通过 Path 2
3. **路径覆盖缺口**:数学定理证明类论文在 CS 中是高质量信号,V11.3 完全没有对应路径

### 3.3 V11.4 方案(N2-A)

**修改文件**:`echelon/seeds/physical_depth.py`

**Path 2 重构为 4 个子路径**:

| 子路径 | 检测目标 | 关键正则 / 条件 |
|---|---|---|
| Path 2a | 性能数字(配对) | SOTA%/F1/AUC/BLEU/ROUGE + 数据集名必须同句出现 |
| Path 2b | 消融研究 | `ablat\w*` + count_word ≥ 3 |
| Path 2c | 复杂度证明 | `O(n)` / `time complexity` / `polynomial` / `convergence rate` |
| Path 2d | 数据集规模 | M+/B+ samples / parameters / FLOPs 数量级词 |

**新增 Path 4 — 理论深度**:

检测条件:`theorem` / `proof` / `lemma` / `we prove` / `bound is tight` 任意 ≥ 2 次出现。

**参数调整**:

- V11.4: `path_count = 4`(Path 1 / Path 2[含 2a-2d] / Path 3 / Path 4)

## 4. N3 — cocite 自适应分位数阈值

### 4.1 现象引用

来源:`V11_4_plan.md` §N3。

```
新语料(2024-2026):cocite 边 56,308,过滤前后几乎无变化
老语料(2022-2023):cocite 原 72,945 → 过滤后 41,718,去噪 42.8%
固定阈值 ≥2 在两个语料的效果差异极大
```

### 4.2 根因

老论文积累引用时间长,共被引矩阵稠密,weight=1 的"巧合共被引"数量庞大。新论文共被引矩阵稀疏,weight=1 反而代表真实的语义关联信号。用统一固定阈值无法适应两种情况:

- 固定 threshold=2 对老语料:合理去噪
- 固定 threshold=2 对新语料:过度去噪,丢弃真实信号

### 4.3 V11.4 方案(N3-A)

**修改文件**:`echelon/graph/cocite.py`

**新函数**:`compute_cocite_threshold(weight_distribution)`

```python
def compute_cocite_threshold(weight_distribution):
    p50 = np.percentile(weight_distribution, 50)
    threshold = max(2, int(p50))
    return threshold
```

含义:以所有共被引对 weight 的 P50 作为阈值候选,取 max(2, P50) 保证下限为 2。

| 语料场景 | weight 分布 P50 | 自适应阈值 |
|---|---|---|
| 新语料(稀疏) | 1 | max(2, 1) = 2 |
| 老语料(稠密) | 2~3 | max(2, 2) = 2 或 max(2, 3) = 3 |
| 合并 2000 篇 | 2 | max(2, 2) = **2** |

**新增可观测字段**:`cocite_threshold_used` 记录在 L1 stats 中。

**V11.4 实际结果**:
- 合并语料 P50 = 2(见 `l1_graph_stats_v4.json` `cocite_distribution.p50 = 2`)
- 阈值 = max(2, 2) = **2**
- co_citation_all_pairs = 249,953 → co_citation_after_filter = **129,857**(过滤 48.1%)

## 5. N4 — c_venue percentile-by-age

### 5.1 现象引用

来源:`V11_4_plan.md` §N4。

```
V11.3 log 空间修复后:top10_range = 0.0447(1.6×,未达 2× 目标)
根本问题:老论文 cited_by_count 累积均质化 c_venue 分量
```

老语料论文 cited_by_count 动辄 ≥100,归一化后全部落在 0.7-0.9 区间,失去区分度。新论文 cited_by_count 0-50,归一化后落在 0-0.5 区间。两个语料的 c_venue 无可比性。

### 5.2 根因

原 c_venue 公式:

```
c_venue = cited_by_count / max(corpus_cited)
```

这个公式用全局最大值做归一化,对老论文"均质化":1000 篇老论文里 cited_by_count 最大可能是 500,则 cited_by=100 的论文得 0.2,cited_by=450 的论文得 0.9——但两者都是同代际中高引论文,差异被放大了。反过来新论文之间的细微差异被压缩。跨语料合并后,新老论文不在同一尺度上比较。

### 5.3 V11.4 方案(N4-A)

**修改文件**:`echelon/seeds/score_keystone.py`

**新公式**:`c_venue_v4(paper, corpus)`

```python
def c_venue_v4(paper, corpus):
    age_months = max(1, (today - paper.publication_date).days / 30)
    cite_per_year = paper.cited_by_count / (age_months / 12)
    peer_papers = [p for p in corpus if abs(p.age_months - age_months) <= 6]
    peer_cites_per_year = [p.cited_by_count / (p.age_months / 12) for p in peer_papers]
    percentile = sum(1 for c in peer_cites_per_year if c <= cite_per_year) / len(peer_papers)
    return percentile  # 0.0 ~ 1.0
```

核心变化:改用"同年龄段(±6 个月)论文中的年化引用率百分位",消除跨时代累积效应。

文档更新:第 4 章 §4.2.3 c_venue 公式重写。

## 8. 已修复 Bug:co_citation 语料内 vs 含语料外(28× 数据修正)

### 8.1 Bug 描述

**Bug 编号**: V11.4-BugFix-1  
**发现时间**: V11.4 子代理上一轮运行  
**错误数据**: co_citation = **4,603**  
**修正数据**: co_citation = **129,857**  
**数据偏差**: 修正后是错误值的 **28.2 倍**

### 8.2 根因分析

co_citation 的语义定义是:"两篇论文被同一第三方论文引用"。此处的第三方论文不限于语料内部,而是包含任意外部论文。

**Bug 原因**:上一轮实现中,co_citation 仅统计了"语料内部相互引用"的共被引对——即只有当第三方引用者同样在 2000 篇语料里才被计入。这与 co_citation 的语义定义相悖。

**正确实现**:扫描每篇论文的 references 字段,对所有(论文 A, 论文 B)对,若存在语料外论文 C 同时引用了 A 和 B,则建立 co_citation 边。语料外引用关系通过 OpenAlex API 的 `referenced_works` 字段反向查找。

### 8.3 影响分析

| 指标 | 错误值(bug) | 修正值 | 倍数 |
|---|---|---|---|
| co_citation 边数 | 4,603 | 129,857 | **28.2×** |
| 总图边数 | ~109,000(估算) | 234,172 | 2.1× |
| co_citation 占总边比例 | ~4.2% | 55.5% | — |

从数据角度,co_citation 作为语义邻近信号(共同被引者的研究者认可重叠),应当是图中边数最多的关系类型之一。修正后的 129,857 与 bib_couple(97,053)同量级,符合工程预期。4,603 仅相当于 cite_direct(4,608)的数量级——在没有引用过滤的情况下,这个数字是完全不合理的。

### 8.4 修正操作

**修改文件**:`echelon/graph/cocite.py`

- 修改 `build_cocite_graph()` 中的查询逻辑:从"语料内引用矩阵"扩展为"含语料外 references 的反向查找"
- 新增 `co_citation_all_pairs` 字段记录过滤前总量,`co_citation_after_filter` 记录过滤后数量

**已修正字段**(`l1_graph_stats_v4.json`):

```json
"co_citation_after_filter": 129857,
"co_citation_all_pairs": 249953
```

**bug 注释**:

```json

```

### 8.5 对三轮对比的影响

V11.2 报告中无 co_citation 精确值记录(V11.2 未启用过滤),V11.3 使用老语料过滤后值 41,718。V11.4 修正后 129,857 大幅提升的原因包含两个独立因素:

1. **语料规模翻倍**:2000 篇 vs 1000 篇,co_citation 对数对理论上 4× 增长
2. **Bug 修复**:从仅计算语料内引用切换到含语料外引用,数量级跳变

两个因素叠加,导致最终倍数约为 28×。
