## 4. 关键洞察

### O1 [bug 已修] co_citation 语料内 vs 含语料外 = 28× 数据丢失

**描述**: V11.4 子代理上一轮运行中,co_citation 计算逻辑存在实现 bug:仅统计了"语料内部两篇论文互相引用某第三篇语料内论文"的共被引对,而忽略了语料外部引用关系。

**错误数据**: co_citation = 4,603  
**修正数据**: co_citation = 129,857  
**倍数**: 129,857 / 4,603 = **28.2×**

**影响范围**:
- 上一轮图谱中 co_citation 边严重欠缺,图连通度受损
- 基于 co_citation 的 bridging_centrality 计算可能存在偏差
- L2 漏斗中依赖图特征的评分可能受影响

**修复状态**: 已修复。本报告所有数据均来自修正后的 `l1_graph_stats_v4.json`。

**注释确认**:
```json
"_v11_4_bugfix_1_note": "co_citation 从错误的 4603(只算语料内引用)修正为 129857(包含语料外引用,符合 co_citation 的语义)"
```

### O2 bib_couple 暴涨 97,053 = 跨时代合并的真实信号(不是 bug)

**数据**:

| 轮次 | bib_couple | 语料 |
|---|---|---|
| V11.2 | 26,764 | raw 1000 (2024-2026) |
| V11.3 | 37,093 | raw_v2 1000 (2022-2023) |
| V11.4 | **97,053** | merged 2000 (2022-2026) |

bib_couple(书目耦合)的语义:两篇论文共同引用了同一参考文献。在时间跨度更大的合并语料中,2022-2023 和 2024-2026 的论文大量共引经典基础文献(BERT、ResNet、GPT-2 等),形成跨时代的书目耦合关系。

**为何是真实信号而非 bug**:
1. bib_couple 边数理论上随语料扩展而超线性增长(经典文献被更多论文共引)
2. 跨时代论文研究同一领域时必然共引领域奠基文献
3. 97,053 = V11.2 (26,764) + V11.3 (37,093) + 跨时代新增 (33,196)——三部分加和合理
4. bib_couple 不依赖引用时间窗,不受"新论文未被引"问题影响

**工程含义**: bib_couple 在跨时代合并场景中是图连通度的主力保障,应保留。V11.5 需增加 bib_couple 边上限防止图稀释(见 §6 遗留问题)。

### O3 ROBOTICS_ML 桥词主导(146 篇 / 2,190 边占 83%)— 数据特性 + Topic 重叠

**数据**:
- 全部 bridge 命中论文:175 篇
- ROBOTICS_ML 命中:146 篇(83.4%)
- ROBOTICS_ML bridge 边:2,190 / 总 bridge 边 2,625 = **83.4%**

**原因分析**:

1. **2022-2023 语料密度效应**: 2022-2023 是深度强化学习应用于机器人操作的爆发期,该时间窗内 RL+Robotics 交叉论文密度远高于其他类别。raw_v2 的 1000 篇包含大量 imitation learning / sim-to-real / behavior cloning 类论文,与 20 条 ROBOTICS_ML 桥词高度匹配。

2. **Topic 结构重叠**: 本轮 4 topic 中有 2 个直接涉及 Robotics(T10653 / T10462),合计占语料 50%(1000 篇)。ROBOTICS_ML 桥词命中这两个 topic 下的论文时,覆盖面天然更广。

3. **不是 bug**: ROBOTICS_ML 桥词命中论文均为真实跨界论文(在机器人领域应用 ML 方法),不是假阳性。146 篇中 VLA / RT-2 / diffusion policy 等词的命中论文均属于该领域核心工作。

**已知风险**: ROBOTICS_ML 主导可能导致图中 Robotics 子图内部连通性过强,相对弱化其他 topic 的桥接结构。V11.5 需添加 ROBOTICS_ML 桥词上限或按 topic 均衡采样。

### O4 N4 top10_range 未达 2× 目标 = MMR 后 high-percentile 论文挤压 → P1 换 SPECTER2 后预期解决

**详细机制**:

KeystoneScore top10 的论文由 MMR(λ=0.7)从 206 篇双门通过论文中选出 100 篇,再取前 10 名。MMR 的多样性惩罚项(max_penalty = 0.335)会主动降低与已选论文相似度高的候选分数。这产生了两个效应:

1. 真正的"金种子"论文(高 percentile c_venue,高 bridging_centrality)之间往往嵌入接近(同 topic 内标杆论文语义相似),被 MMR 相互惩罚
2. 最终 top10 中的论文分数集中在 [0.665, 0.715] 区间(range = 0.0472),而非期望的 [0.5, 1.0] 宽区间

**为何 P1 换 SPECTER2 预期解决**:
- SPECTER2(768D)的嵌入空间比当前 all-MiniLM-L6-v2(384D)更丰富
- 同 topic 论文在 SPECTER2 空间中区分度更高,MMR 惩罚项更精准
- 高质量论文之间的余弦相似度更低,MMR 不会大幅压降其分数
- 更大的嵌入空间理论上可使 top10_range 扩展到 2× 目标

**当前 PARTIAL 状态不阻塞 P1 启动**:c_venue_std = 0.286 > 0.20 已验证年龄偏置消除,top10_range 问题是嵌入模型限制而非 N4 算法本身问题。

### O5 N2 Path 4 理论深度仅 1.2% = 关键词集偏窄,需 V11.5 扩充

**数据**: Path 4 命中 10 篇(1.2%),目标 ≥5%(≥41 篇)

**当前关键词集**:
```
theorem / proof / lemma / we prove / bound is tight
```

**问题**: 这套词汇覆盖数学/形式化证明风格的论文(ICML/NeurIPS 理论方向),但以下常见 CS 理论论文惯用表达未被覆盖:

| 未覆盖表达 | 代表场景 |
|---|---|
| `we show that` | 几乎所有理论结果引导句 |
| `we analyze` | 分析类论文 |
| `we establish` | 建立界/保证 |
| `sample complexity` | 学习理论核心指标 |
| `regret bound` / `regret analysis` | 在线学习/RL 理论 |
| `PAC learning` | 统计学习理论 |
| `information-theoretic` | 信息论下界 |
| `lower bound` / `upper bound` | 通用理论界 |
| `we prove that` | 证明引导(比 `we prove` 更常见的完整表达) |

**V11.5 计划**: 将 Path 4 关键词扩充到 20 条,预期将命中率从 1.2% 提升至 5-8%。

## 6. 已知遗留问题(列入 V11.5)

### 6.1 Path 4 关键词扩充

**问题**: V11.4 Path 4 理论深度关键词集(5 条)过窄,命中率仅 1.2%。  
**优先级**: P1-fix

2. 目标关键词总数:15-20 条
3. 验收标准:2000 篇语料 Path 4 占比 ≥5%

### 6.2 co_citation 跨代合并阈值动态调整(P50 同质化问题)

**问题**: 合并语料 P50=2,阈值 max(2, P50)=2,与固定阈值等效。N3 方案在当前数据集上未体现自适应优势。

**深层问题**: 合并语料的 co_citation 分布由新老语料混合,P50 稳定在 2,动态调整无法生效。

**V11.5 计划**:
1. 分层计算阈值:V1 语料独立计算 P50,V2 语料独立计算 P50,取较大值
2. 验证分层阈值是否使去噪率更合理
3. 或改用 P60 替代 P50 作为阈值候选

### 6.3 ROBOTICS_ML 桥词上限(防主导)

**问题**: ROBOTICS_ML 占全部 bridge 论文 83%,造成图中 Robotics 子图内部过度连通,可能挤压其他 topic 的桥接权重。

**V11.5 计划**:
1. 在 `bridge_keywords.py` 中添加 `per_category_max_bridges` 参数
2. 建议上限:ROBOTICS_ML ≤ 50%(按 bridge 边数)
3. 超出上限时按 KeystoneScore 排序截取

### 6.4 T11714 主导金种子 → topic-balanced 约束

**问题**: V11.4 金种子 T11714(Multimodal ML) 占 37%,T10245(Metasurfaces)仅 12%,分布失衡。

**数据**:

| Topic | 实际占比 | 期望均匀占比 | 偏差 |
|---|---|---|---|
| T11714 | 37% | 25% | +12% |
| T10653 | 27% | 25% | +2% |
| T10462 | 24% | 25% | -1% |
| T10245 | 12% | 25% | -13% |

**根因**: T11714 论文的语义特征在当前嵌入空间(all-MiniLM)中更集中,KeystoneScore 分布更高,MMR 无法完全均衡。

**V11.5 计划**:
1. 增加 topic-balanced 约束:每 topic 金种子数量不超过 `total / n_topics × 1.5`(即不超过 37.5 篇)
2. 或在 MMR 阶段引入 topic 均衡惩罚项

### 6.5 bib_couple 边上限(防稀释)

**问题**: V11.4 bib_couple = 97,053,占总边(234,172)的 41.4%。随语料规模进一步增大,bib_couple 可能超线性增长并稀释其他边类型的权重。

**V11.5 计划**:
1. 评估 bib_couple 边数上限(建议 ≤ co_citation 边数的 1.5×)
2. 设计 bib_couple 边的权重衰减公式(按共同引用数 top 百分位截断)
3. 添加 `bib_couple_max_edges` 参数到 L1 builder
